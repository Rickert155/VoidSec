# VoidSec

В мире информационной безопасности существует множество дистрибутивов для проведения тестирования на проникновение, анализа безопасности и цифровой криминалистики. Такие дистрибутивы включают в себя, как правило, сотни предустановленных иструментов. 

Однако не всем пользователям нужен специализированный дистрибутив, многие предпочитают использовать для задач свою освную операционную систему, будь то Debian, Arch Linux, Fedora, Void Linux или любой другой дистрибутив на базе ядра Linux. Есть разные способы получить и использовать необходимые инструменты:
- Установка специализованного дистрибутива на виртуальную машину
- Установка специализованного дистрибутива на USB-накопитель
- Установка дополнительных репозиториев(к примеру BlackArch)
- Использование готовых Docker/Podman образов.  

У всех решений есть свои преимущества, но есть и недостатки. Так для установки на виртуальную машину хосту необходимо жертвовать ресурсами, соизмеримыми как для полноценной ОС. У USB-накопителей происходит быстрее износ накопителя из-за постоянной перезаписи. Подключение репозиториев привязывают к конкретным дистрибутивам или их пакетным менеджерам. В готовых Docker/Podman образах же предустановлено большое количество инструментов, которыми пользователь и не будет пользоваться, а вес таких образов может начинаться от 2GB.  

**VoidSec** предлагает другой подход для решения.  
Проект представляет из себя набор готовых Containerfile и package.json с мета-данными для сборки инструмента из исходных текстов в изолированном окружении.  

Цели проекта:
- Независомость от выбора дистрибутива основной ОС;
- Отсутствие конфликтов зависимостей в основной системе;
- Изоляция инструментов друг от друга;
- Стандартизация сборки инструментов в изолированных средах;

VoidSec не заменяет специализированные операционные системы и готовые образы другиех разработчиков и сообщества в целом. VoidSec предлагает альтернативный подход к использованию инструментов в сфере информационной безопасности.
