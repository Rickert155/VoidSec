# VoidSec

В мире информационной безопасности существует множество дистрибутивов для проведения тестирования на проникновение, анализа безопасности и цифровой криминалистики. Такие дистрибутивы включают в себя, как правило, сотни предустановленных иструментов. 

Однако не всем пользователям нужен специализированный дистрибутив, многие предпочитают использовать для задач свою основную операционную систему, будь то Debian, Arch Linux, Fedora, Void Linux или любой другой дистрибутив на базе ядра Linux. Есть разные способы получить и использовать необходимые инструменты:
- Установка специализованного дистрибутива на виртуальную машину;
- Установка специализованного дистрибутива на USB-накопитель;
- Установка дополнительных репозиториев(к примеру BlackArch);
- Использование готовых Docker/Podman образов.  

У всех решений есть свои преимущества, но есть и недостатки. Так для установки на виртуальную машину хосту необходимо жертвовать ресурсами, соизмеримыми как для полноценной ОС. У USB-накопителей происходит быстрее износ накопителя из-за постоянной перезаписи. Подключение репозиториев привязывают к конкретным дистрибутивам или их пакетным менеджерам. В готовых Docker/Podman образах же предустановлено большое количество инструментов, которыми пользователь и не будет пользоваться, а вес таких образов может начинаться от 2GB.  

**VoidSec** предлагает другой подход для решения.  
Проект представляет из себя набор готовых Containerfile и package.json с мета-данными для сборки инструмента из исходных текстов в изолированном окружении.  

Цели проекта:
- Независомость от выбора дистрибутива основной ОС;
- Отсутствие конфликтов зависимостей в основной системе;
- Изоляция инструментов друг от друга;
- Стандартизация сборки инструментов в изолированных средах;

VoidSec не заменяет специализированные операционные системы и готовые образы других разработчиков и сообщества в целом. VoidSec предлагает альтернативный подход к использованию инструментов в сфере информационной безопасности.

## Правила оформления инструментов в рамках проекта **VoidSec**
В директории packages создается директория инструмента.  
В директории пакета создается два файла: *Containerfile* и *package.json*.  

**package.json** должен сожержать в себе следующие обязательные поля:
- name;
- description;
- source;
- containerfile;
- image.  

Пример такого файла:
```json
{
    "name":"cyberscan", # название инструмента
    "description":"Сканер web-приложений", # краткое описание
    "source":"https://gitea.com/cyberwarn/cyberscan", # источник исходных текстов
    "containerfile":"Containerfile", # имя файла для сборки образа
    "image":"localhost/cyberscan-container" # имя образа
}
``` 
Инструкция для сборки образа можно указать отличную от Containerfile. В таком случае в *package.json* необходимо будет указать соответствующее значение для ключа *containerfile*

## VoidSec CLI
**VoidSec CLI** не является оберткой для Podman или чем-то подобный. VoidSec CLI является скриптом для автоматиазации сборки/обновления/удаления образов, установляемых через VoidSec.  

Примеры использовани:  
**Сборка образа**
```sh
python3 -m voidsec --build cyberscan
```
Будет собран образ cyberscan. Запуск скрипта в данном случае равнозначен следующей команде:
```sh
podman build -t cyberscan-container -f packages/cyberscan/Containerfile
```
**Обновление образа**
```sh
python3 -m voidsec --update cyberscan
```
Фактически тоже самое, что и build.  

**Удаление образа**
```sh
python3 -m voidsec --delete cyberscan
```
